The big idea

Keep the largest value at the root

Heap Sort gives guaranteed O(n log n) time and constant extra space, but it is not stable.

i.

Build a heap

Repair subtrees from the middle of the array back to the root.

ii.

Extract max

Swap the root with the last heap item. That final slot is now sorted.

iii.

Heapify down

Move the new root downward until every parent is larger than its children.

When to use Heap Sort

Use Heap Sort when you need guaranteed O(n log n) time and must keep extra memory constant.

  • Embedded or memory-constrained systems.
  • Priority-queue based scheduling concepts.
  • Cases where worst-case bounds matter but stability does not.
