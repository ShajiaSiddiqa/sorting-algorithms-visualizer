The big idea

Sort small runs, then merge them

Merge Sort is stable and predictable. It pays extra memory to guarantee O(n log n) time, even when the input is already awkward.

i.

Split

Keep dividing the range in half until every piece has one element.

ii.

Compare fronts

During merge, only the first unmerged item from each half needs to be compared.

iii.

Write back

The smaller front value is copied into the output position, preserving equal-value order.

When to use Merge Sort

Use Merge Sort when predictable O(n log n) time and stability matter more than extra memory.

  • Sorting linked lists, where merging is cheap.
  • External sorting for files too large to fit fully in memory.
  • Stable ordering in reports, logs, and multi-key data pipelines.
