The big idea

Find the next smallest item

Selection Sort does a predictable number of comparisons and very few swaps, but it does not adapt to nearly sorted data.

i.

Start a slot

The next position in the sorted prefix needs the smallest remaining value.

ii.

Scan for minimum

Walk the rest of the array and update the minimum whenever a smaller value appears.

iii.

Swap into place

Move that minimum into the open sorted slot, then repeat for the next slot.

When to use Selection Sort

Use Selection Sort when writes are expensive and input size is small, because it performs at most one swap per position.

  • Teaching minimum selection and sorted-prefix growth.
  • Small arrays stored on media where writes should be limited.
  • Simple demonstrations of in-place sorting without stability.
